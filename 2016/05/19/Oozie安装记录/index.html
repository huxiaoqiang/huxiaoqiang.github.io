Oozie安装记录

Mac Oozie安装记录

##说明
用本文档记录和总结我在本机上安装Oozie的过程，作为后面复习总结的参考

Oozie环境依赖

  • Unix box (tested on Mac OS X and Linux)
  • Java JDK 1.6+
  • Maven 3.0.1+
  • Hadoop 0.20.2+
  • Pig 0.7+
  • ExtJS library (optional, to enable Oozie webconsole)

###本机环境

  • Mac OS 10.11.4 2.2GHz Intel Core i7
  • Java:1.8.0 91 JRE:1.8.0_91-b14
  • Apache Maven 3.3.9

Java安装以及环境变量设置

hadoop需要java版本在1.6及以上，在terminal中通过java -version，显示本机的java版本：

java version "1.8.0_91"
Java(TM) SE Runtime Environment (build 1.8.0_91-b14)
Java HotSpot(TM) 64-Bit Server VM (build 25.91-b14, mixed mode)

Maven安装

Oozie依赖Maven，要求版本在3.0.1及以上，配置安装过程如下：

下载

下载 Maven, 并解压到目录 /usr/local/apache-maven-3.3.9

###设置环境变量
vi ~/.bash_profile打开terminal 添加：

export M2_HOME=/usr/local/apache-maven-3.3.9
export M2=$M2_HOME/bin
export PATH=$PATH:$M2

保存后执行命令 source ~/.bash_profile 让bash_profile生效。

在terminal中通过mvn -version,显示如下，表明Maven安装成功。

ExtJS library安装

ExtJS 2.2

Hadoop安装配置，以及测试

###准备条件 ssh keys
除了java环境的配置，还需要SSH keys。首先需要打开远程登录（Remote Login）,可以在 系统偏好设置->共享->远程登录 勾选 。通过ssh localhost，如果提示输入账号密码，说明ssh设置成功，如果没有，通过 

ssh-keygen -t rsa -P ""

设置，并且避免每次输入密码，授权到本地：

cat $HOME/.ssh/id_rsa.pub >> $HOME/.ssh/authorized_keys

下载安装

通过homebrew安装：

brew install hadoop

下载完后，在路径 /usr/local/中下载到Cellar文件夹，具体目录是/usr/local/Cellar/hadoop/2.7.2

环境配置

hadoop的所有组件都是通过xml文件来配置，路径如下：/usr/local/Cellar/hadoop/2.7.2/libexec/etc/hadoop，MapReduce配置在 mapred-site.xml，HDFS配置在 hdfs-site.xml，公共参数在 core-site.xml 中，hadoop环境参数在 hadoop-env.sh 中。
需要的配置参数如下：

hadoop-env.sh

在 export HADOOP_OPTS=下面添加如下：

export HADOOP_OPTS="$HADOOP_OPTS -Djava.security.krb5.realm=OX.AC.UK -Djava.security.krb5.kdc=kdc0.ox.ac.uk:kdc1.ox.ac.uk"

core-site.xml

note:fs.default.name当前设置为localhost是因为本地调试的目的，如果在网络中设置多节点，需要设置为：hdfs://<ComputerName>.local:9000。获取你本机的名字，可以通过系统偏好设置（System Preferences） -> 共享（Sharing），获取本机名。

<configuration>
    <property>
        <name>hadoop.tmp.dir</name>
           <value>/tmp/hadoop-${user.name}</value>
           <description>A base for other temporary directories.</description>
     </property> 
    <property>
           <name>fs.default.name</name>
           <value>hdfs://localhost:9000</value>
     </property>
</configuration>

hdfs-site.xml

Hadoop Distributed File System，Hadoop分布式文件系统参数，配置参数在这个文件中，配置如下：

<configuration>
    <property>
           <name>dfs.replication</name>
        <value>1</value>
      </property>
</configuration>

mapred-site.xml

mapreduce配置在这个文件中，配置如下：

<configuration>
    <property>
           <name>mapred.job.tracker</name>
        <value>localhost:9001</value>
      </property>
</configuration>

###基本安装成功
开始运行daemons之前，需要先格式化新安装的HDFS，格式化通过创建存储路径和初始元数据来创建一个新的文件系统。

hadoop namenode -format

确认依然登录在localhost，start-dfs.sh 启动HDFS，$ start-mapred.sh 启动map-reduce，start-all.sh启动所有。

监控：

HDFS Administrator : http://localhost:50070

Task Tracker : http://localhost:50060

MapReduce Administrator : http://localhost:50030

pig安装

###下载
在http://mirror.bit.edu.cn/apache/pig/中下载相应版本，比如pig-0.15.0，下载解压到 /usr/local/中

###环境变量设置
vi ~/.bash_profile在bash_profile中添加

export PIG_HOME=/usr/local/pig-0.15.0
export PATH=$PATH:$PIG_HOME/bin

source ~/.bash_profile，让bash_profile立即生效，在terminal输入pig -help，查看pig是否可用。
