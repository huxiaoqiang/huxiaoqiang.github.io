Oozie安装

首先总结一下本次安装Oozie依赖的版本

  • Java JDK 1.7.0_79
  • Oozie 4.2.0
  • Maven 3.3.9
  • Hadoop 2.6.0
  • Pig 0.15.0
    这里需要说明的是，JDK不能是1.8，1.8版本的java在build时，不能编译通过，Hadoop的版本参考网上的2.6.0，2.7版本的，我自己尝试了，编译的过程会出错，具体原因不详，我也是遇到很多坑了，后面才退到2.6.0版本，本地编译通过，没有问题。

Oozie安装配置文档介绍了Oozie安装配置的详细参数，从Oozie-4.2.0下载Oozie4.2.0。根据官网的介绍，Oozie安装过程分为下面三步

building Oozie

官网上介绍，通过bin/mkdistro.sh是最简单的build方式，但是不能直接简单运行，因为pom.xml中许多默认的软件版本，和我下载的版本不一样，需要做相应的改动，首先是修改目标java版本，如下，将targetJavaVersion改为1.7。注意，不能用java1.8，Oozie应该不支持java1.8版本，build的过程会出错

然后进入到oozie/bin目录下，执行下面命令进行编译。
参数说明，由于hadoop使用的是2.6.0版本，所以需要参数hadoop-2，表明是hadoop版本2，其他的可选参数还有：hadoop-1、hadoop-3分别表示版本1和3， hadoop-0.23表示hadoop0.23。
由于Oozie4.2.0默认的hadoop-2是指的2.3.0，所以需要在后面指定hadoop的具体版本，`-Dhadoop.auth.version=2.6.0 -Ddistcp.version=2.6.指定hadoop具体版本为2.6.0。最后是pig版本，本机装的是0.15.0，直接指定就行。

另外，根据需要，还可以指定如下相关软件版本：

  • Dsqoop.version= - default 1.4.3
  • Dsqoop.classifier= - default hadoop100
  • Dtomcat.version= - default 6.0.41
  • Dopenjpa.version= - default 2.2.2
  • Dxerces.version= - default 2.10.0
  • Dcurator.version= - default 2.5.0
  • Dhive.version= - default 0.13.1
  • Dhbase.version= - default 0.94.2
